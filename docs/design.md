# WinPulse 设计文档

作者：Zoran B [https://github.com/2017-z]
版本：0.1
日期：0-26-2026

## 设计目标

WinPulse 是一个低侵入、后台运行、以日志为核心的 Windows 系统性能监控工具。
  则该工具的设计在目前阶段遵循最小开销原则，应当尽可能减少UI的设计，不应当追求实时UI

## 使用场景
- 用户希望应用不在前台的时候，依然能够监控到系统性能
- 用户需要自主排查特定时段的系统性能问题
- 用户希望长期了解系统资源使用趋势

## 功能需求
- 后台运行
- 定时采样系统资源
- 生成日志文件
- 支持自定义采样频率
- CPU、内存、磁盘、显卡等资源监控
  CPU 使用率、内存使用率 & 可用内存、磁盘读写速率、GPU 使用率 & 显存占用
- Top N 内存占用进程
- 系统启动时间、当前时间戳

## 约束与边界
- 不提供图形界面，日志文件输出到本地
- 不需要支持多实例运行
- 当前版本不涉及内核驱动
- 运行于Windows用户态
- 当前版本不支持跨平台
- 在采样频率为 1s 时，工具自身 CPU 使用率应小于 0.5%
- 常驻内存（RSS）应控制在 15MB 以内
- 当某个性能计数器不可用（如显卡驱动异常）时，程序不应崩溃，而应在日志中记录缺失标识

## 技术选型
- 语言标准：C++20
- 构建系统：CMake 3.20.0
